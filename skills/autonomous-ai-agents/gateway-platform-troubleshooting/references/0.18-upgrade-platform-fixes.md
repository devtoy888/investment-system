# Hermes 0.18 → 0.18.2 Platform Dependency Fixes

## Summary of Breaking Changes

After upgrading to Hermes 0.18.2, these platform adapters broke:

| Platform | Error | Root Cause | Fix |
|----------|-------|------------|-----|
| Feishu | `Client.__init__() got an unexpected keyword argument 'extra_ua_tags'` | lark-oapi 1.5.3 doesn't have `extra_ua_tags` param; 0.18.2 adapter code uses it | Upgrade lark-oapi to 1.6.8 |
| Telegram | `'HTTPXRequest' object attribute 'do_request' is read-only` | httpx 0.28.1 made `do_request` read-only; python-telegram-bot 22.6 patches it at init | Downgrade httpx to 0.27.2 |
| Config | `providers.custom: unknown config keys ignored: agnes` | `providers.custom.agnes` nested format returns empty from `providers_dict_to_custom_providers()` | Change to flat format: `providers.agnes:` |

## Diagnostic Commands

### Check lark-oapi version
```bash
# In .feishu-deps (persistent, may be stale)
cat /opt/data/.feishu-deps/lark_oapi-*.dist-info/METADATA | grep '^Version:'
# In lazy-packages (persistent, may be stale)
cat /opt/data/lazy-packages/lark_oapi-*.dist-info/METADATA | grep '^Version:'
# In Hermes venv (rebuilt on upgrade, should be current)
cat /opt/hermes/.venv/lib/python3.13/site-packages/lark_oapi-*.dist-info/METADATA | grep '^Version:'
# Check what version the adapter expects
grep 'lark-oapi==' /opt/hermes/tools/lazy_deps.py
```

### Check httpx version
```bash
# Current version
python3 -c 'import httpx; print(httpx.__version__)'
# Expected by Hermes (from pyproject.toml)
grep 'httpx' /opt/hermes/pyproject.toml
```

### Verify providers format
```python
from hermes_cli.config import providers_dict_to_custom_providers

# Nested format (broken in 0.18.2):
providers_dict_to_custom_providers({"custom": {"agnes": {"base_url": "...", "key_env": "...", "model": "..."}}})
# Returns: [] (empty!)

# Flat format (correct):
providers_dict_to_custom_providers({"agnes": {"base_url": "...", "key_env": "...", "model": "..."}})
# Returns: [{'name': 'agnes', 'base_url': '...', ...}] (correct)
```

## Fix Steps

### 1. Fix lark-oapi (Feishu)
```bash
# Remove stale copies
rm -rf /opt/data/.feishu-deps/lark_oapi*
rm -rf /opt/data/lazy-packages/lark_oapi*

# Install to Hermes venv
pip install lark-oapi==1.6.8

# Copy to .feishu-deps (so the .pth chain finds it)
cp -r /opt/hermes/.venv/lib/python3.13/site-packages/lark_oapi* /opt/data/.feishu-deps/

# Verify
python3 -c "from lark_oapi.ws import Client; import inspect; print('extra_ua_tags' in inspect.signature(Client.__init__).parameters)"
# Expected: True
```

### 2. Fix httpx (Telegram)
```bash
# Downgrade in Hermes venv
pip install 'httpx[socks]==0.27.2'

# Also fix in .feishu-deps (which may shadow the venv)
rm -rf /opt/data/.feishu-deps/httpx*
cp -r /opt/hermes/.venv/lib/python3.13/site-packages/httpx* /opt/data/.feishu-deps/

# Verify
python3 -c "import httpx; print('httpx:', httpx.__version__)"
# Expected: httpx: 0.27.2
```

### 3. Fix providers format (config.yaml)
In each profile's config.yaml, change:
```yaml
# Before (broken):
providers:
  custom:
    agnes:
      base_url: https://apihub.agnes-ai.com/v1
      key_env: AGNES_API_KEY
      model: agnes-2.0-flash

# After (correct):
providers:
  agnes:
    base_url: https://apihub.agnes-ai.com/v1
    key_env: AGNES_API_KEY
    model: agnes-2.0-flash
```

Also update fallback references:
```yaml
# Before:
provider: custom:agnes
# After:
provider: agnes
```

### 4. Restart
```bash
docker restart hermes-main
# Or for individual gateways:
printf r > /run/service/gateway-default/supervise/control
```

## Key Files

| File | Location | Purpose |
|------|----------|---------|
| `.feishu-deps/` | `/opt/data/.feishu-deps/` | Persistent package store, added to sys.path via `.pth` |
| `lazy-packages/` | `/opt/data/lazy-packages/` | Alternative lazy install target |
| `lark_oapi` | `ws/client.py` | WebSocket client with `extra_ua_tags` param (1.6.8+) |
| `hermes_feishu_deps.pth` | Hermes venv site-packages | Points to `.feishu-deps` directory |
| `lazy_deps.py` | `/opt/hermes/tools/lazy_deps.py` | Lazy install specs (line 191: `lark-oapi==1.6.8`) |
