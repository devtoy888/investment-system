# 基金推送格式指南

> 建立于：2026-06-30 用户反馈累积
> 适用于三条推送的 cron prompt 格式模板设计

## 收盘复盘格式要求

### 1. 方向验证 — 比较早盘预测 vs 收盘实际涨跌

**临界修正（2026-06-30）：** 验证的不是"开方向vs收方向"，而是"早盘预测的方向 vs 收盘实际涨跌幅"。

❌ 错误逻辑：开4058↓ → 收4094↑ → ❌（这是低开高走，不是预测验证）

✅ 正确逻辑：
| 指数 | 开方向预测 | 收盘实际 | 验证 |
|:----|:---------:|:--------:|:----:|
| 上证指数 | ↑ 预计高开 | +0.50%↑ | ✅ 预测涨实际涨 |
| 科创50 | ↑ 预计高开 | +3.85%↑ | ✅ 预测涨实际涨 |
| 黄金ETF | ↓ 预计低开 | -0.84%↓ | ✅ 预测跌实际跌 |

**数据来源：** `_morning_predictions.json`（今日参考 cron 的 LLM 在推送前保存的）

如果该文件不存在（首次部署第一天），prompt 必须写"PASS：如果文件不存在则跳过"，不能编造数据。

### 2. 今日判断验证 — 必须逐项对比 + 含昨收列

用户明确要求不能只写一个总结分数。必须列出每个指数的早盘方向 vs 收盘方向。**涨跌是从昨收算的，不是从今开算的**，表格必须有昨收列避免混淆。

❌ 错误写法（无昨收列，用户会误解）：
```
| 指数 | 今开 | 收盘 | 涨跌 |
|------|:---:|:----:|:----:|
| 黄金ETF | 8.645 | 8.750 | 🟢 -0.84% |
```
（收8.750 > 开8.645但涨跌是负的，因为-0.84%是从昨收8.824算的）

✅ 正确写法（含昨收列）：
```
| 指数 | 昨收 | 今开 | 收盘 | 涨跌 |
|------|:---:|:---:|:----:|:----:|
| 黄金ETF | 8.824 | 8.645 | 8.750 | 🟢 -0.84% |
```

### 3. 板块排行 — 必须展示今开今收

```
| 板块 | 今开→收盘 | 涨跌 |
|------|:--------:|:----:|
| 通信 | 1.380→1.397 | 🔴 +4.33% |
```

### 4. 基金持仓 — 每支基金单独一行表格

❌ 错误写法（塞进分组行）：
```
科技/AI: 🔴 +3.20% (7支) [明细...]
```

✅ 正确写法（每个基金单独一行）：
```
【科技/AI】
| 基金 | 昨收净值 | 今日估算 | 涨跌 |
|------|:-------:|:-------:|:----:|
| 华夏科创50ETF联接C | 1.6477 | 1.7111 | 🔴 +3.85% |
```

### 5. 北向资金 — 带时间戳

```
| 渠道 | 净流入 |
|------|:------:|
| 沪股通 | 🟢 -9.28亿 |
| 深股通 | 🔴 -31.10亿 |
| 合计 | 🟡 -40.38亿 |
(时间: 15:00)
```

## 推送自检模板

每条推送prompt最后一个步骤必须是**格式自检**，检查项如下：

```
## 第X步：格式自检
推送前执行以下核验：
1. 表格列对齐：列数与模板一致，每行都有数据
2. 数据一致性：所有数据与预采集文件一致
3. 预测来源：早盘预测必须从 _morning_predictions.json 读取
4. 涨跌幅带正负号 +/-
5. 禁止出现禁用词（月/季/半年末日不可写"明日"）
6. 非交易日则跳过，不编造行情数据
7. 表格格式：不能退化为纯文字堆叠
```

## 特殊日期措辞

| 日期 | 含义 | 推演措辞 |
|------|------|---------|
| 6/30 | 上半年收官日 | "下一个交易日（下半年首个交易日）" **不说"明日"** |
| 3/31 | 一季度收官日 | "二季度首个交易日" |
| 9/30 | 三季度收官日 | "四季度首个交易日" |
| 12/31 | 全年收官日 | "明年首个交易日" |
| 月末 | 月线收盘 | "下月首个交易日" |

## 数据异常标注规则

- status=⚠️ 时，推送开头追加 `⚠️ 本日数据异常：[warnings前2条]`
- status=✅ 时，不追加标注

## 三条推送通用规则

1. 🔴=涨 🟢=跌（国内A股习惯）
2. 表格必须有表头，列必须对齐
3. 数据为空或不可用则跳过，不展示空行/空表
4. 纯文本（不需要MEDIA图片）
5. deliver 到 qqbot
6. 每次修改prompt后，必须用 session_search 验证最新推送格式未退化
