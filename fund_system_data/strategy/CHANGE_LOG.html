2. 微博内容分析框架 v2（重点）

2.1 之前的问题

问题 表现 根因
内容截断 最长仅171字 text[:500] + 短预览API
无事实核查 静态关键词匹配 ClaimVerifier无法匹配赛道→行情

时序不分 今日/本周/长期混为一谈 无timeframe筛选

2.2 v2框架设计

4层架构，每层解决一个具体问题：

┌─ 信号提取层(Extractor) ─────────────────────────┐
│  输入: KOL博文文本                                  │
│  输出: {赛道, 方向, 时效, 断言原文, 置信度}         │
│  改进: 区分 today/soon/long 三个时间窗口            │
└────────────────────────────────────────────────────┘
                          ↓
┌─ 事实核查层(Verifier) ────────────────────────────┐
│  输入: 信号 + 腾讯行情数据                           │
│  输出: ✅ 预测方向与市场一致 / ❌ 不一致             │
│  改进: 赛道→行情key映射表（科技/AI→科创50/半导体）   │
└────────────────────────────────────────────────────┘
                          ↓
┌─ 操作映射层(Mapper) ─────────────────────────────┐
│  输入: 已核查的信号                                  │
│  输出: {赛道, 操作方向, 具体基金, 仓位建议}          │
│  改进: 赛道→基金映射（科技/AI→011613/017103..）     │
└────────────────────────────────────────────────────┘
                          ↓
┌─ 格式化输出层(format_push) ──────────────────────┐
│  输出: QQ Bot Markdown → 今日操作建议表格            │
│        → 信号摘要(含✅/❌标记) → 统计汇总           │
└────────────────────────────────────────────────────┘
