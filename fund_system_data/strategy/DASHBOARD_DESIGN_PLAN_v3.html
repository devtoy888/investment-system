Worker示例

export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    const db = env.DB;

    if (path === '/api/dashboard') {
      const { results } = await db.prepare(
        `SELECT * FROM portfolio_snapshots WHERE date = date('now')`
      ).all();
      return Response.json(results);
    }

    if (path.startsWith('/api/history')) {
      const days = url.searchParams.get('days') || 30;
      const { results } = await db.prepare(
        `SELECT date, total_value, total_profit_pct
         FROM portfolio_snapshots WHERE date >= date('now', '-' || ? || ' days')
         ORDER BY date`
      ).all(days);
      return Response.json(results);
    }

    return env.ASSETS.fetch(request);
  }
}

6.2 数据流三层架构

第一层 JSONL（本地磁盘，唯一真相源）
  cron写, 前端不写. 12个文件, 包括signals-resolved.jsonl(953条)

第二层 D1（Cloudflare SQLite，查询加速）
  sync_to_d1.py定期同步, 支持SELECT查询
  D1挂了→数据安全（真相源在JSONL）

第三层 R2 JSON（CDN缓存，兜底）
  cron每次推送后写dashboard.json
  前端: 优先D1 → 降级R2 → 极端降级空状态
