# 多智能体（Swarm）与 Shadow Account

## Swarm 多智能体交易团队

### 架构

Swarm 系统基于 **DAG（有向无环图）执行引擎**，支持预设的研究团队协作流程。

### 30 个预设团队

| 预设 | 成员 | 职责 |
|------|------|------|
| **investment_committee** | macro, quant, risk, catalyst | 投资委员会 - 综合评审 |
| **quant_desk** | 量化分析师 | 策略开发和因子分析 |
| **risk_review** | 风控专家 | 风险审查和压力测试 |
| **macro** | 宏观分析师 | 宏观环境分析 |
| **crypto** | 加密研究员 | 加密货币市场分析 |
| **value_investing_committee** (v0.1.11) | 价值投资团队 | 基本面 + 报告审计 |

### Swarm 执行流程
1. 选择预设或自定义团队
2. Worker 并行获取实时市场数据作为依据
3. 按 DAG 顺序执行分析
4. 流式 SSE 进度报告
5. 输出聚合报告（含证据、指标、注意事项）

### CLI 使用

```bash
# 启动投资委员会
vibe-trading --swarm-run investment_committee --goal "分析美联储降息对 A 股的影响"

# 启动价值投资委员会（v0.1.11）
vibe-trading --swarm-run value_investing_committee --goal "评估茅台当前估值"
```

### 内容过滤弹性（v0.1.10+）
Swarm 运行跳过个别 LLM 内容审核命中，在 Run Card 中记录高过滤率警告。

### Swarm API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/swarm/presets` | GET | 列出所有预设 |
| `/swarm/runs` | POST | 启动 Swarm 运行 |
| `/swarm/runs/{id}/events` | GET | SSE 事件流 |

## Shadow Account（影子账户）

### 核心概念
Shadow Account 不是模拟账户，而是**行为诊断工具**——解析你的交易日志，提取你实际使用的隐式规则，然后跑反事实回测，看看"如果你严格按照这些规则交易会怎样"。

### 工作流程

```
交易日志 → 规则提取 → 策略生成 → 反事实回测 → 审计报告
```

### 4 步流程

1. **解析交易日志** — 支持券商导出文件（CSV 等）
2. **提取隐式规则** — Agent 分析你的买卖行为，提取规则（含 RSI 范围、前 N 日收益等条件）
3. **生成策略代码** — 基于规则自动生成可运行的策略代码
4. **反事实回测** — 在历史数据上重新执行策略，对比实际交易结果

### CLI 使用

```bash
# 分析交易日志
vibe-trading --upload trades_export.csv
vibe-trading run -p "Profile my trading behavior and identify any biases"

# 完整 Shadow Account 流程
vibe-trading run -p "Extract my trading rules and run a shadow backtest"
```

### 技术细节

- **PIT 安全**的入场上下文 — `entry_rsi14` 和 `prior_5d_return` 通过加载器注册表在 `buy_dt` 时获取
- 条件入场边界（v0.1.10+）— 生成的 SignalEngine 在真实条件（RSI 在范围内、前 N 日收益在范围内）入场，而非盲目复现持仓节奏
- `PRICE_FEATURES` 共享契约 — 规则提取和代码生成用同一份合约，防止漂移

### 审计报告
- 行为偏差分析
- 规则一致性检查
- 策略代码导出（可修改和重新回测）

### MCP 暴露的工具
`analyze_trade_journal`、`extract_shadow_strategy`、`run_shadow_backtest`、`render_shadow_report`、`scan_shadow_signals`
