# GB/T 28448-2019 网络安全等级保护测评要求

> **标准全称**：信息安全技术 网络安全等级保护测评要求  
> **英文名称**：Information security technology — Evaluation requirement for classified protection of cybersecurity  
> **标准号**：GB/T 28448-2019（替代 GB/T 28448-2012）  
> **发布机构**：国家市场监督管理总局、中国国家标准化管理委员会  
> **发布日期**：2019-05-10  
> **实施日期**：2019-12-01  
> **页数**：291 页  
> **篇幅**：约 262,000 字符  

---

## 一、标准概述

GB/T 28448-2019 是 **GB/T 22239-2019《网络安全等级保护基本要求》** 的配套测评标准，规定了不同安全保护等级信息系统的 **测评方法、测评内容、测评判定标准**。两份标准构成等保 2.0 的核心"双标"体系：

| 标准号 | 角色 | 内容 |
|--------|------|------|
| **GB/T 22239-2019** | 建设/整改依据 | 规定各级系统"应该做到什么"（安全要求） |
| **GB/T 28448-2019** | 测评/验收依据 | 规定各级系统"如何验证做到了"（测评方法） |

该标准适用于：
- 测评机构对等级保护对象的安全测评
- 运营使用单位对等级保护对象的**自测评**
- 主管部门对等级保护对象的**监督检查**
- 网络安全监管部门的**行政执法**

---

## 二、五级保护等级体系

标准覆盖 **5 个安全保护等级**，各级测评要求逐级递增：

| 等级 | 名称 | 安全防护定位 | 测评要求特点 |
|:----:|------|------------|------------|
| **第一级** | 自主保护级 | 公民、法人和其他组织合法权益受侵害，但不会损害国家安全、社会秩序和公共利益 | 基础测评项，主要依赖访谈和文档审查 |
| **第二级** | 指导保护级 | 公民、法人和其他组织合法权益受到严重侵害，或社会秩序和公共利益受到侵害 | 增加配置核查，覆盖常规安全防护能力 |
| **第三级** | 监督保护级 | 社会秩序和公共利益受到严重侵害，或国家安全受到侵害 | 增加工具检测和实地查验，全面深度测评（最常见等级） |
| **第四级** | 强制保护级 | 国家安全受到严重侵害 | 增加高强度对抗测试，要求更严格的安全机制验证 |
| **第五级** | 专控保护级 | 国家安全受到特别严重侵害 | 最高级别，含定制化、国家级安全保障措施验证 |

> **测评实践中**：第三级是最常见的等级，覆盖政务、金融、能源、交通、水利、教育、医疗等关键信息基础设施。

## 三、10 大安全维度（安全类）

每个等级均覆盖 **10 个安全类**，每个安全类下分若干 **安全子类（控制点）**，构成 `5 等级 × 10 维度 = 50 个测评单元矩阵`。

### 3.1 安全大类概览

| 序号 | 安全类 | 英文 | 核心关注 | 主要测评方式 |
|:----:|--------|------|---------|------------|
| 1 | **安全物理环境** | Physical Security | 机房、设备间、布线等物理环境的安全性 | 访谈 + 实地查验 |
| 2 | **安全通信网络** | Communication Network Security | 网络架构、通信传输、可信验证 | 配置核查 + 工具检测 |
| 3 | **安全区域边界** | Boundary Security | 边界防护、访问控制、入侵防范、安全审计 | 配置核查 + 工具检测 |
| 4 | **安全计算环境** | Computing Environment Security | 身份鉴别、访问控制、安全审计、入侵防范、数据完整性/保密性、数据备份恢复、剩余信息保护、个人信息保护、可信验证 | 访谈 + 配置核查 + 工具检测 |
| 5 | **安全管理中心** | Security Management Center | 系统管理、审计管理、安全管理、集中管控 | 访谈 + 配置核查 |
| 6 | **安全管理制度** | Security Management Policy | 安全策略、管理制度、制定和发布、评审和修订 | 访谈 + 文档审查 |
| 7 | **安全管理机构** | Security Management Organization | 岗位设置、人员配备、授权和审批、沟通和合作、审核和检查 | 访谈 + 文档审查 |
| 8 | **安全管理人员** | Personnel Security | 人员录用、离岗、安全意识教育和培训、外部人员访问管理 | 访谈 + 文档审查 |
| 9 | **安全建设管理** | Security Construction Management | 安全方案设计、产品采购和使用、自行软件开发、外包软件开发、工程实施、测试验收、系统交付、等级测评、服务供应商管理 | 访谈 + 文档审查 |
| 10 | **安全运维管理** | Security Operation Management | 环境管理、资产管理、介质管理、设备维护、漏洞和风险管理、网络和系统安全管理、恶意代码防范管理、配置管理、密码管理、变更管理、备份与恢复管理、安全事件处置、应急预案管理、外包运维管理 | 访谈 + 文档审查 + 配置核查 |

### 3.2 各级安全类覆盖情况

```
              L1    L2    L3    L4    L5
安全物理环境    ✓     ✓     ✓     ✓     ✓
安全通信网络    ✓     ✓     ✓     ✓     ✓
安全区域边界    ✓     ✓     ✓     ✓     ✓
安全计算环境    ✓     ✓     ✓     ✓     ✓
安全管理中心    -     -     ✓     ✓     ✓
安全管理制度    ✓     ✓     ✓     ✓     ✓
安全管理机构    ✓     ✓     ✓     ✓     ✓
安全管理人员    ✓     ✓     ✓     ✓     ✓
安全建设管理    ✓     ✓     ✓     ✓     ✓
安全运维管理    ✓     ✓     ✓     ✓     ✓
```

> 注：第一级和第二级不要求"安全管理中心"，第三级起为必须。

## 四、4 大扩展安全场景

除上述 10 个通用安全类外，标准还规定了 **4 种扩展场景的安全测评要求**，适用于特定技术环境：

### 4.1 云计算安全扩展要求

| 维度 | 测评项关注点 |
|------|-----------|
| 安全物理环境 | 云计算基础设施物理位置、机房环境 |
| 安全通信网络 | 虚拟网络隔离、网络架构与VPC验证 |
| 安全区域边界 | 虚拟化边界防护、虚拟机间流量监控 |
| 安全计算环境 | 镜像和快照保护、多租户隔离、数据迁移保护 |
| 安全管理中心 | 云管理平台安全管理、审计数据集中 |
| 安全建设管理 | 云服务商选择与合同、供应链安全 |
| 安全运维管理 | 云服务商运维能力、SLA 验证 |

**责任划分模式**：标准区分"基础设施位于境内/境外""IaaS/PaaS/SaaS"等不同服务模式下测评对象的差异。

### 4.2 移动互联安全扩展要求

| 维度 | 测评项关注点 |
|------|-----------|
| 安全物理环境 | 移动终端物理安全（丢失/被盗应对） |
| 安全通信网络 | 无线通信加密、传输完整性 |
| 安全区域边界 | 移动接入网关、VPN验证 |
| 安全计算环境 | 移动应用安全（签名、沙箱、权限）、移动终端管理（MDM/EMM） |

### 4.3 物联网安全扩展要求

| 维度 | 测评项关注点 |
|------|-----------|
| 安全物理环境 | 感知节点物理防护（防破坏、防盗窃） |
| 安全通信网络 | 感知层网络通信安全、异构网络接入 |
| 安全区域边界 | 感知节点接入控制 |
| 安全计算环境 | 感知节点设备安全（固件完整性、安全启动） |
| 安全运维管理 | 感知节点生命周期管理（入网、运行、退网） |

### 4.4 工业控制系统安全扩展要求

| 维度 | 测评项关注点 |
|------|-----------|
| 安全物理环境 | 工业控制设备物理环境适应性（温湿度、电磁防护） |
| 安全通信网络 | 工业控制网络分层分域、实时性要求验证 |
| 安全区域边界 | 办公网与控制网边界隔离、无线工业网络控制 |
| 安全计算环境 | 控制设备安全（PLC/DCS/SCADA）、工业协议安全 |
| 安全建设管理 | 工业控制系统安全方案、供应链管理 |
| 安全运维管理 | 工业控制系统运维（不停机维护、应急响应） |

## 五、测评方法体系

GB/T 28448-2019 定义了 **五种核心测评方法**，构成测评工作的完整方法论：

### 5.1 测评方法汇总

| 方法 | 编号 | 说明 | 适用对象 | 典型工具/手段 |
|------|:----:|------|---------|-------------|
| **访谈** | ① | 通过与相关人员交谈，了解安全措施落实情况 | 管理制度、人员管理、安全意识 | 访谈提纲、录音记录 |
| **文档审查** | ② | 查阅策略文件、制度文件、运维记录等文档材料 | 管理制度、建设管理、运维管理 | 检查表、对照清单 |
| **配置核查** | ③ | 登录系统查看安全配置是否合规 | 网络设备、安全设备、服务器、数据库 | 核查脚本、配置基线 |
| **工具检测** | ④ | 使用专用安全工具进行技术测试 | 漏洞扫描、渗透测试、代码审计 | Nessus/OpenVAS/AppScan/Nmap |
| **实地查验** | ⑤ | 到物理现场实地查看安全防护措施 | 机房环境、物理设备、布线 | 拍照/摄像、查验记录表 |

### 5.2 各级测评方法权重

不同等级中各类测评方法的比重不同：

| 等级 | 访谈权重 | 文档审查权重 | 配置核查权重 | 工具检测权重 | 实地查验权重 |
|:----:|:-------:|:----------:|:----------:|:----------:|:----------:|
| 一级 | ★★★★★ | ★★★★★ | ★★☆ | ☆☆☆☆ | ★★★☆ |
| 二级 | ★★★★☆ | ★★★★☆ | ★★★★ | ★★★☆ | ★★★☆ |
| 三级 | ★★★☆☆ | ★★★☆☆ | ★★★★★ | ★★★★★ | ★★★★☆ |
| 四级 | ★★★☆☆ | ★★★☆☆ | ★★★★★ | ★★★★★ | ★★★★★ |
| 五级 | ★★★☆☆ | ★★★☆☆ | ★★★★★ | ★★★★★ | ★★★★★ |

> 高等级更依赖技术层面验证（配置核查+工具检测），低等级更侧重管理层面（访谈+文档审查）。

### 5.3 测评流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  测评准备    │ → │  方案编制    │ → │  现场测评    │ → │  报告编制    │
│  - 信息收集  │    │  - 测评指标  │    │  - 访谈      │    │  - 单项判定  │
│  - 对象确定  │    │  - 测评方法  │    │  - 文档审查  │    │  - 单元判定  │
│  - 工具准备  │    │  - 工具选择  │    │  - 配置核查  │    │  - 整体判定  │
│              │    │  - 进度安排  │    │  - 工具检测  │    │  - 整改建议  │
│              │    │              │    │  - 实地查验  │    │              │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 六、测评判定标准

### 6.1 判定层级

标准采用 **三级判定体系**：

```
单项判定（控制点项级）
    ↓ 汇总
单元判定（安全类/控制点级）
    ↓ 汇总
整体判定（系统级）
    ↓ 输出
测评结论（符合/基本符合/不符合）
```

### 6.2 单项判定规则

对每个 **测评项（控制点下的具体测评要求）** 的判定：

| 判定结果 | 判定条件 |
|:--------:|---------|
| **符合** | 测评对象满足该项所有要求，且运行有效 |
| **部分符合** | 测评对象基本满足要求，但存在部分不足或运行不够有效 |
| **不符合** | 测评对象不满足要求，或者未实施 |
| **不适用** | 该项要求因测评对象实际环境不适用（需说明理由并在报告中注明） |

### 6.3 单元判定规则

将同一安全类下的所有测评项结果汇总：

| GB/T 22239-2019 要求 | 单元判定 | 说明 |
|:--------------------:|:--------:|------|
| 所有受控测评项均为"符合" | **符合** | 该安全类完全满足要求 |
| 所有受控测评项均为"符合"或"部分符合" | **基本符合** | 该安全类基本满足要求，存在部分不足 |
| 任一受控测评项为"不符合" | **不符合** | 该安全类存在不满足项 |

### 6.4 整体判定规则

| 判定结果 | 判定条件 |
|:--------:|---------|
| **符合** | 所有安全类均为"符合" |
| **基本符合** | 所有安全类均为"符合"或"基本符合"，且安全风险可控 |
| **不符合** | 任一安全类为"不符合"，或安全风险不可控 |

### 6.5 特殊判定规则

- **高风险项一票否决**：若存在可能导致系统被轻易攻破或数据大规模泄露的 **关键测评项** 为"不符合"，即使其他项全为"符合"，整体判定仍可为"不符合"。
- **不适用项处理**：不适用项不参与判定，但必须由测评机构在报告中逐项说明不适用的理由和技术依据。
- **补偿措施**：被测单位可提供补偿性安全措施，经测评机构评估后，可替代某项要求的达到效果，判定为"符合"。

## 七、与 GB/T 22239-2019 的对照关系

GB/T 28448-2019 的每个测评项都 **一一对应** GB/T 22239-2019 的安全要求项：

### 7.1 对照表示例（第三级—安全计算环境—身份鉴别）

| GB/T 22239-2019 要求 | GB/T 28448-2019 测评方法 | 预期结果 |
|----------------------|-------------------------|---------|
| 应对登录的用户进行身份标识和鉴别 | ① 访谈系统管理员，了解身份鉴别措施；② 核查系统身份鉴别功能配置 | 系统启用了用户名+口令/证书/生物特征等身份鉴别机制 |
| 身份标识应唯一 | ③ 核查是否存在同名用户或共享账户 | 所有账户标识唯一，无共享账户 |
| 身份鉴别信息应具有复杂度要求 | ③ 核查口令策略配置（长度、复杂度、更换周期） | 符合复杂度策略要求 |
| 应具有登录失败处理功能 | ③ 核查账户锁定策略；④ 使用工具模拟多次失败登录 | 达到阈值后账户锁定 |
| 当进行远程管理时，应采取措施防止鉴别信息被窃听 | ③ 核查远程管理是否使用SSH/HTTPS等加密协议；④ 使用抓包工具验证 | 鉴别信息传输过程加密 |

### 7.2 各级测评项数量分布

| 等级 | 通用安全类测评项数 | 扩展场景测评项数 | 合计（约） |
|:----:|:---------------:|:-------------:|:--------:|
| 第一级 | ~38 | — | ~38 |
| 第二级 | ~138 | — | ~138 |
| 第三级 | ~226 | +~84（含4类扩展） | ~310 |
| 第四级 | ~252 | +~92 | ~344 |
| 第五级 | ~260+ | +~100+ | ~360+ |

## 八、与 GB/T 28448-2012 的主要变化

| 变化维度 | GB/T 28448-2012（旧） | GB/T 28448-2019（新） |
|---------|---------------------|---------------------|
| **适用对象** | 仅信息系统 | 信息系统 + 云计算/移动互联/物联网/工业控制 |
| **安全类数量** | 10 类（技术 5 + 管理 5） | 10 通用类 + 4 扩展场景 |
| **测评方法** | 访谈、检查、测试 | 访谈、文档审查、配置核查、工具检测、实地查验（5 种） |
| **可信计算** | 未涉及 | 增加可信验证要求 |
| **个人信息保护** | 未涉及 | 增加个人信息保护测评项 |
| **大数据/新技术** | 未涉及 | 覆盖云计算等新技术场景 |
| **判定规则** | 较简略 | 三级判定体系（单项→单元→整体），带高风险一票否决 |
| **与基本要求对照** | 存在不对应 | 严格一一对照 GB/T 22239-2019 |
