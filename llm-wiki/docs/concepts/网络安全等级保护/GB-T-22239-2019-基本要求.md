# GB/T 22239-2019 信息安全技术 网络安全等级保护基本要求

## 一、标准概述

GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》是**网络安全等级保护制度（等保2.0）**的核心标准之一，由国家市场监督管理总局和中国国家标准化管理委员会于2019年5月10日发布，2019年12月1日正式实施，替代了旧版 GB/T 22239-2008。

该标准规定了网络安全等级保护的第一级到第四级等级保护对象的基本安全要求，适用于指导分等级的非涉密对象的安全保护建设和监督管理。相比等保1.0，等保2.0在保护对象、保护内容和覆盖范围上进行了全面升级：

- **保护对象扩展**：从传统的信息系统扩展到云计算、移动互联、物联网、工业控制系统等新型应用场景。
- **安全要求重构**：从原有的10个安全类扩展为**安全通用要求**和**安全扩展要求**两大体系。
- **可信计算引入**：在各安全等级中逐步引入可信验证要求，实现从被动防御向主动免疫的转变。
- **"一个中心、三重防护"**：以安全管理中心为支撑，构建安全计算环境、安全区域边界、安全通信网络三重防护体系。

## 二、十个安全维度（安全通用要求）

GB/T 22239-2019 将安全通用要求划分为**10个安全类**，覆盖技术和管理两大层面：

| 序号 | 安全类 | 英文/缩写 | 分类 | 核心内容概述 |
|:---:|-------|----------|:---:|------------|
| 1 | **安全物理环境** | Physical Security | 技术 | 机房选址、温湿度控制、电力供应、防盗窃防破坏、防火防水防雷、电磁防护等物理层面的安全要求 |
| 2 | **安全通信网络** | Communication Network Security | 技术 | 网络架构设计、通信传输加密、可信验证等，确保数据在网络传输过程中的机密性和完整性 |
| 3 | **安全区域边界** | Boundary Security | 技术 | 边界防护、访问控制、入侵防范、恶意代码防范、安全审计、可信验证等，构建网络边界的纵深防御 |
| 4 | **安全计算环境** | Computing Environment Security | 技术 | 身份鉴别、访问控制、安全审计、入侵防范、恶意代码防范、数据完整性与保密性、数据备份恢复、可信验证等 |
| 5 | **安全管理中心** | Security Management Center | 技术 | 系统管理、审计管理、安全管理和集中管控，实现"一个中心"的集中安全管理能力 |
| 6 | **安全管理制度** | Security Management Policy | 管理 | 安全策略制定、管理制度体系建设、安全策略的制定和发布、评审和修订机制 |
| 7 | **安全管理机构** | Security Management Organization | 管理 | 岗位设置、人员配备、授权和审批流程、沟通协作机制、审核和检查制度 |
| 8 | **安全管理人员** | Personnel Security | 管理 | 人员录用与离岗管理、安全意识教育培训、外部人员访问管理、保密协议签订 |
| 9 | **安全建设管理** | Security Construction Management | 管理 | 系统定级备案、安全方案设计、产品采购使用、自行/外包软件开发、工程实施、测试验收、系统交付、等级测评和服务供应商管理 |
| 10 | **安全运维管理** | Security Operation Management | 管理 | 环境管理、资产管理、介质管理、设备维护、漏洞和风险管理、网络和系统安全管理、恶意代码防范、密码管理、变更管理、备份与恢复、安全事件处置、应急预案和外包运维管理 |

> **说明**：前5项（安全物理环境、安全通信网络、安全区域边界、安全计算环境、安全管理中心）属于**技术类**要求，后5项（安全管理制度、安全管理机构、安全管理人员、安全建设管理、安全运维管理）属于**管理类**要求，体现了"技术与管理并重"的等保核心理念。

## 三、五级安全保护等级要求

GB/T 22239-2019 依据等级保护对象受到破坏后对公民、法人和其他组织的合法权益、社会秩序和公共利益及国家安全的危害程度，将安全保护等级划分为五级：

### 第一级（自主保护级）

- **侵害客体**：公民、法人和其他组织的合法权益
- **侵害程度**：造成损害
- **监管强度**：自主保护
- **核心要求**：提供最基本的保护措施，包括简单的身份鉴别、自主访问控制和基本的物理环境保护。无需配置安全管理中心，安全审计要求较宽松。

### 第二级（指导保护级）

- **侵害客体**：公民、法人和其他组织的合法权益；或社会秩序和公共利益
- **侵害程度**：造成严重损害；或造成危害
- **监管强度**：指导保护
- **核心要求**：在第一级基础上显著增强。要求实施强制访问控制、完整的安全审计功能（审计记录需保护）、入侵防范和恶意代码防范机制、数据完整性和保密性保护。需建立基本的安全管理体系，包括管理制度、管理机构和人员管理。

### 第三级（监督保护级）⭐ 最常见

- **侵害客体**：社会秩序和公共利益；或国家安全
- **侵害程度**：造成严重损害；或造成危害
- **监管强度**：监督保护
- **核心要求**：等保测评中最常见的等级。在第二级基础上全面强化，要求双因子身份鉴别、基于安全标记的强制访问控制、全面的安全审计（审计记录需保护且定期备份）、入侵防范和恶意代码防范的集中管理、全面的数据加密和备份恢复机制。**强制要求建立安全管理中心**，实现系统管理、审计管理、安全管理和集中管控的"一个中心"体系。需定期开展等级测评。

### 第四级（强制保护级）

- **侵害客体**：社会秩序和公共利益；或国家安全
- **侵害程度**：造成特别严重损害；或造成严重损害
- **监管强度**：强制保护
- **核心要求**：在第三级基础上进一步强化，要求更严格的身份鉴别机制（如多因子鉴别）、全面的可信验证、更高级别的加密算法、异地实时备份和灾难恢复能力。安全管理中心实现全面的集中管控和态势感知。安全管理制度和管理体系更加严密。

### 第五级（专控保护级）

- **侵害客体**：国家安全
- **侵害程度**：造成特别严重损害
- **监管强度**：专控保护
- **核心要求**：最高安全保护等级，适用于涉及国家核心秘密和重大国家安全利益的系统。GB/T 22239-2019 仅对第五级提出原则性要求，具体安全要求由相关主管部门另行制定。

### 各级要求对照表

| 要求项 | 第一级 | 第二级 | 第三级 | 第四级 |
|:------|:-----:|:-----:|:-----:|:-----:|
| 身份鉴别 | ✓ 基本 | ✓✓ 增强 | ✓✓✓ 双因子 | ✓✓✓✓ 多因子 |
| 访问控制 | 自主 | 强制 | 标记+强制 | 标记+强制+可信 |
| 安全审计 | 简单 | ✓ 完整 | ✓✓ 保护+备份 | ✓✓✓ 实时告警 |
| 入侵防范 | — | ✓ | ✓✓ 集中管理 | ✓✓✓ 主动防御 |
| 恶意代码防范 | — | ✓ | ✓✓ 集中管理 | ✓✓✓ 综合防护 |
| 数据加密 | 可选 | 传输加密 | 存储+传输 | 全生命周期 |
| 备份恢复 | 基本 | 本地备份 | 异地备份 | 实时异地+灾备 |
| 安全管理中心 | — | 可选 | ✓ 强制 | ✓✓ 全面 |
| 可信验证 | — | — | 可基于可信根 | ✓ 全程可信 |

## 四、四种安全扩展场景

GB/T 22239-2019 针对新技术、新应用场景，在安全通用要求之外，特别规定了**4种安全扩展要求**，以适应不同形态等级保护对象的安全需求：

### 4.1 云计算安全扩展要求

云计算安全扩展要求适用于使用云计算技术构建的等级保护对象，覆盖 **IaaS、PaaS、SaaS** 三种服务模式以及 **公有云、私有云、混合云** 等部署模式。

| 扩展项 | 核心要求 |
|-------|---------|
| **"云-租"责任共担** | 明确云服务商与云租户各自的安全责任边界，服务商负责"云"本身的安全（基础设施、虚拟化层），租户负责"云"上的安全（应用、数据、身份管理） |
| **基础设施位置** | 要求云计算基础设施位于中国境内，确保数据和业务的合规性 |
| **虚拟化安全** | 虚拟机隔离与安全防护、虚拟网络边界防护、虚拟机镜像和快照的安全保护 |
| **多租户隔离** | 不同租户的计算资源、存储资源和网络资源的有效隔离 |
| **数据安全** | 云上数据存储位置的可知性、数据迁移和删除的彻底性、数据加密和密钥管理 |
| **运维管理** | 云平台管理面的安全、远程运维的安全管控 |
| **安全审计** | 支持云租户对自身资源的独立审计能力、审计数据的隔离 |

### 4.2 移动互联安全扩展要求

移动互联安全扩展要求适用于使用移动终端通过无线方式接入网络的等级保护对象，覆盖**移动终端、移动应用和无线网络**三个层面。

| 扩展项 | 核心要求 |
|-------|---------|
| **移动终端管控** | 终端设备注册管理、终端安全策略（如禁用摄像头、蓝牙、USB接口等）、终端环境安全检测 |
| **移动应用安全** | 移动应用软件的安全审核与签名、应用权限最小化、应用数据本地存储加密 |
| **无线接入安全** | 无线接入设备的物理安全、无线信号的覆盖范围控制、无线通信加密（WPA2/WPA3等） |
| **边界防护** | 无线网络与有线网络之间的访问控制和隔离、非法无线设备的检测与阻断 |
| **移动终端准入** | 基于设备身份、用户身份、安全状态的准入控制机制 |

### 4.3 物联网安全扩展要求

物联网安全扩展要求适用于使用物联网技术的等级保护对象，覆盖**感知层、网络层和应用层**三层架构，主要针对各类传感器、RFID标签、智能终端等感知设备。

| 扩展项 | 核心要求 |
|-------|---------|
| **感知节点物理安全** | 感知节点设备的物理防护（防盗窃、防破坏、防非法替换）、环境适应性要求 |
| **感知节点接入安全** | 感知节点接入网络的认证与授权机制，防止非法节点接入和伪造 |
| **感知网络传输安全** | 异构网络环境下的通信加密、数据完整性校验 |
| **网关节点安全** | 物联网网关的访问控制、安全审计、入侵防范，作为感知层和网络层的安全桥梁 |
| **感知数据安全** | 感知数据的完整性、保密性保护，数据防篡改和抗重放攻击 |
| **设备管理** | 物联网设备的统一管理、状态监控、固件安全更新、异常行为检测 |

### 4.4 工业控制系统安全扩展要求

工业控制系统安全扩展要求适用于工业控制系统（ICS），包括 **SCADA、DCS、PLC、SIS** 等工业自动化控制系统，遵循"安全分区、网络专用、横向隔离、纵向认证"的十六字安全防护原则。

| 扩展项 | 核心要求 |
|-------|---------|
| **室外控制设备防护** | 室外工控设备（如RTU、PLC等）的物理安全防护，防止非法物理接触和破坏 |
| **工业控制网络架构** | 生产控制大区与管理信息大区的安全分区、工业控制网络与企业网络的物理/逻辑隔离 |
| **拨号使用控制** | 远程拨号访问的严格管控，仅在必要时启用并进行全程审计 |
| **无线使用控制** | 工控环境中无线通信的严格控制，避免对控制信号的干扰和未经授权的接入 |
| **控制设备安全** | 控制设备（PLC、DCS控制器等）的访问控制、安全审计、固件完整性保护、配置变更管理 |
| **实时性保障** | 安全措施不得影响工业控制系统的实时性和可用性（可用性优先原则） |
| **安全监控** | 工业控制网络的安全监测与异常行为分析，支持工业协议的深度解析 |

> **重要提示**：工业控制系统安全以**可用性为第一优先级**，其次才是完整性和机密性。这与传统信息系统以机密性优先的安全策略有本质区别。
