# Dynamic Related Pages (Client-Side Injection)

Client-side JS approach to auto-display related wiki pages derived from Graphify's knowledge graph. Injects a `📊 图谱关联` section dynamically on every page, showing the top N most-connected neighbor pages.

## Why Client-Side Over Pre-Injection

| Approach | Pros | Cons |
|----------|------|------|
| Python pre-injection (append `📊 图谱关联` to every .md) | Visibly baked into source | Requires re-run after every graph rebuild. Idempotency check needed. Edits .md files. |
| **Client-side JS (recommended)** | Zero write to .md files. Works on graph rebuild immediately. One script serves all pages. | Requires graph.json to be fetchable. Slight delay on page load. |

## Architecture

```
graphify-out/graph.json   →   cp to   →   docs/data/graph/graph.json
                                                    ↓
docs/javascripts/related-pages.v4.js  ← fetches ← ─┘
    │
    └─→ reads h1 title → matches node in graph → finds neighbors → renders links
    └─→ fallback: URL-based matching → same pipeline
```

## Key Components

### 1. URL Mapping Table (PAGE_URLS)

A hardcoded mapping from wiki page name (as a path key) to its canonical URL:

```javascript
var PAGE_URLS = {
  "setup-guide": "/setup-guide/",
  "log": "/concepts/vibe-trading/log/",
  "SCHEMA": "/SCHEMA/",
  "概述": "/concepts/网络安全等级保护/概述/",
  "项目总览": "/concepts/vibe-trading/项目总览/",
  // ... every wiki page
};
var INDEX_URLS = {
  "root": "/",
  "queries": "/queries/",
  "entities": "/entities/",
  // ... every index directory
};
```

**How to generate**: Either manually maintained when pages are created, or auto-generated by a build script that walks the docs directory tree.

### 2. h1 Title Matching (Primary)

Matches the current page by its `<h1>` text content against graph.json node labels:

```javascript
var pageTitle = document.querySelector("h1");
var titleText = pageTitle ? pageTitle.textContent.replace(/\u00b6\s*$/, "").trim() : "";
```

**Critical: MkDocs adds `¶` (pilcrow, U+00B6)** to h1 elements via TOC permalink. The `.replace(/\u00b6\s*$/, "")` strips it before matching against graph node labels.

```javascript
// Match by iterating fileNodes (grouped by source_file)
var curFile = "";
if (titleText) {
  for (var sf in fileNodes) {
    for (var ni = 0; ni < fileNodes[sf].length; ni++) {
      if (fileNodes[sf][ni].label === titleText) {
        curFile = sf;
        break;
      }
    }
    if (curFile) break;
  }
}
```

### 3. URL-Based Fallback (Secondary)

If h1 matching fails, fall back to PAGE_URLS / INDEX_URLS lookup:

```javascript
if (!curFile) {
  for (var fn in PAGE_URLS) {
    if (currentPath === PAGE_URLS[fn] || currentPath.indexOf(fn) !== -1) {
      curFile = fn + ".md";
      break;
    }
  }
}
// Then INDEX_URLS for directory pages
if (!curFile) {
  for (var dir in INDEX_URLS) {
    if (currentPath === INDEX_URLS[dir]) {
      curFile = "index.md";
      break;
    }
  }
}
```

### 4. Graph-File Matching (Tertiary)

Last resort: iterate graph.json's source_file values and compute a candidate URL:

```javascript
if (!curFile) {
  for (var sf in fileNodes) {
    var candidate = "/" + sf.replace(/\.md$/, "") + "/";
    if (currentPath === candidate || currentPath + "/" === candidate) {
      curFile = sf;
      break;
    }
  }
}
```

### 5. Neighbor Discovery & Dedup

Finds all neighbor nodes via graph edges, then maps file-to-file relations:

```javascript
// Get current node IDs
var curIds = {};
fileNodes[curFile].forEach(function(n) { curIds[n.id] = true; });

// Find related nodes via links
var related = {};
links.forEach(function(l) {
  var isSrc = curIds[l.source];
  var isTgt = curIds[l.target];
  if (isSrc || isTgt) {
    var otherId = isSrc ? l.target : l.source;
    if (!curIds[otherId]) {
      if (!related[otherId]) related[otherId] = { count: 0, weight: 0 };
      related[otherId].count++;
    }
  }
});
```

**Dedup by page URL** — prevent the same page appearing twice (via two different node IDs pointing to the same source_file):

```javascript
// Find source files for related nodes
var fileScores = {};
Object.keys(related).forEach(function(nid) {
  nodes.forEach(function(n) {
    if (n.id === nid && n.source_file) {
      if (!fileScores[n.source_file]) fileScores[n.source_file] = { count: 0, names: {} };
      fileScores[n.source_file].count += related[nid].count;
      fileScores[n.source_file].names[n.label || n.id] = true;
    }
  });
});

// Sort by count, pick top N
var sorted = Object.keys(fileScores).sort(function(a, b) {
  return fileScores[b].count - fileScores[a].count;
});

// Render links (dedup by seen URL)
var seen = {};
sorted.slice(0, 8).forEach(function(sf) {
  var url = PAGE_URLS[sf.replace(/\.md$/, "")] || "/" + sf.replace(/\.md$/, "") + "/";
  if (seen[url]) return;  // ← dedup fix
  seen[url] = true;
  // Create <li><a href="...">...</a></li>
});
```

### 6. Rendering to DOM

The script looks for a `📊 图谱关联` h2 section already in the page markup (see markdown template below). It finds the adjacent `<ul>` element and populates it:

```javascript
var headings = document.querySelectorAll("h2");
var section = null;
for (var hi = 0; hi < headings.length; hi++) {
  if (headings[hi].textContent.indexOf("图谱关联") !== -1) {
    section = headings[hi];
    break;
  }
}
if (!section) return;

var list = section.nextElementSibling;
while (list && list.tagName !== "UL") {
  list = list.nextElementSibling;
}
if (!list || list.dataset.relatedLoaded) return;
list.dataset.relatedLoaded = "1";  // prevent double-load
```

### 7. The `.md` Markup (Template)

Each wiki page should include the target section:

```markdown
## 📊 图谱关联

<!-- JS fills this dynamically -->
```

The script detects the h2 text `图谱关联`, finds the next `<ul>`, and fills it with `<li><a href="...">Page Name</a></li>` entries.

## CDN Cache Busting

Cloudflare caches `.js` files for 4 hours. After updating `related-pages.js`, the old version continues to be served:

```bash
# Create versioned copy
cp docs/javascripts/related-pages.js docs/javascripts/related-pages.v4.js

# Update reference in mkdocs.yml
sed -i 's|related-pages.v3.js|related-pages.v4.js|' mkdocs.yml

# Restart container
docker restart llm-wiki
```

Each version bump (v3 → v4 → v5) creates a new URL Cloudflare hasn't cached.
