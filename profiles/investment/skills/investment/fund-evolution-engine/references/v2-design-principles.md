# v2 分析引擎设计原则

## 头号原则：LLM不是美工，是首席分析师

| ❌ v1 错误用法（已弃用） | ✅ v2 正确用法（当前标准） |
|:------------------------|:--------------------------|
| 250-400字输出限制 | **无限字数**，800-2500字每个分析 |
| 传截断数据 `data[:300]` | 传**完整数据**，让LLM自行判断重点 |
| 单次flat prompt | **多步推理链**：画像→赛道→逐基→风险 |
| 无置信度 | **每个判断带置信度1-10** |
| 无预测回溯 | **先验证昨天预测准不准**再分析今天 |
| 没持仓深度 | **自动扫描操作记录**构建真实持仓 |
| 没KOL交叉验证 | **引用KOL观点**做交叉验证 |
| 孤立分析 | **跨报告上下文**：知道上午/昨天说了什么 |

## 四维数据完整性检查

**每次LLM分析前，必须确保以下4项数据全部就绪：**

| 维度 | 数据源 | 采集函数 | 传入方式 |
|:----|:-------|:---------|:--------|
| **① KOL观点** | `_kol_summary.txt` / `_noon_kol.txt` / `fund_tools.get_user_weibos()` | `get_kol_today()` | `━━━ 标题 ━━━` 格式段落 |
| **② 多日趋势** | daily-snapshots.jsonl (5日指数) | `get_multi_day_trend(5)` | 表格：日期/上证/科创50/沪深300/创业板/上证50/黄金 |
| **③ 持仓盈亏** | daily-snapshots.jsonl (成本+市值+盈亏率) | `get_portfolio_pnl()` | 总成本5996元/市值5603元/-393元(-6.55%) |
| **④ 市场新闻** | news_sources.json → RSS采集(requests+xml) | `get_news_headlines(6-8)` | 标签化：🤖AI ⚡能源 🏛️宏观 🥇黄金 |

**实现关键**：数据在 `build_xxx_data_v2()` 中构建为完整段落，LLM自行判断哪些数据重要。

## 7种报告 → 基金推荐覆盖矩阵

每个报告必须包含**操作建议**（不仅是行情分析）：

| 报告 | 推荐深度 | 评分维度 |
|:----|:--------|:--------|
| 早报 (09:00) | 关注3支+警惕3支+其余中性 | 简易优先级 |
| 午报 (11:35) | 赛道级方向+异动观察 | 赛道级方向 |
| **决策 (14:30)** | **全部14支逐基评分+建议** | **技术分+资金分+趋势分** |
| 收盘 (16:00) | 赛道方向+部分基金级 | 明日方向参考 |
| 周度 (周日) | 每支基金调仓+仓位幅度 | 策略级配比建议 |
| 周末 (周六) | 赛道影响+情景预案 | 策略级 |

**决策任务评分系统**（输出表格格式）：
```
| 代码 | 简称 | 赛道 | 技术分 | 资金分 | 趋势分 | 总分 | 操作 | 理由 | 置信度 |
```
- 技术分(1-10)：当日涨跌+量价+位置
- 资金分(1-10)：北向+成交额
- 趋势分(1-10)：多日趋势方向
- 总分 = (技术+资金+趋势) / 3

## 5个v2提示词模板（`llm_analysis_v2.py`）

### 决策任务（14:30 — 最关键，含评分系统）
```
Step1: 全市场画像 + 5日趋势定位 + 新闻驱动
Step2: 分赛道多空诊断（含KOL交叉验证）
Step3: 逐基金评分+操作建议 → 14支逐支三维度评分
Step4: 风险检查清单 → 逐项标记✅/❌
```

### 收盘复盘
```
Step1: 市场定性 → Step2: 板块轮动解构
→ Step3: 组合逐基诊断 → Step4: 关键信号识别
→ Step5: 明日推演（3情景+概率+应对）
→ Step6: 明日基金操作方向参考
```

### 早盘（09:00）
```
Step1: 隔夜信号传导链 → Step2: 昨日复盘+预测验证
→ Step3: 今日作战地图(3条主线) → Step4: 风险清单
→ Step5: 今日基金关注优先级(3关注+3警惕+中性)
```

### 午盘（11:35）
```
Step1: 上午vs昨日预测验证 → Step2: 半日轮动识别
→ Step3: 量价深度分析 → Step4: 午后策略调整
→ Step5: 午后基金操作方向(赛道级)
```

### 周度复盘
```
Step1: 市场全景叙事 → Step2: 组合归因分析
→ Step3: KOL准确率评估 → Step4: 下周基金调仓(配比+幅度)
```

## 数据传递规范

不要截断。全量数据传给LLM自行判断重点：

```python
# ❌ 错误（v1）
data = table[:200] + kol[:300]  # 数据截断

# ✅ 正确（v2）
data = table + kol + fivetrend + pnl + news  # 完整数据(6000+字符)
```

## 数据质量审计

LLM分析前应检查数据源可用性和新鲜度：
- `_source_availability.jsonl` — 各数据源最近可用率
- `_tag_freshness()` 的 `_fresh`/`_stale` 标记
- 当数据源失败时，分析中注明"该数据暂缺"

目前尚未嵌入到v2流程中，需通过 `get_data_quality_report()` 函数补充。

## Cron路径规范

```python
# 数据采集+格式化 → /opt/data/scripts/ 下
# cron wrapper → ~/profiles/investment/scripts/run_xxx.py
# LLM分析模块 → /opt/data/scripts/llm_analysis_v2.py
# 输出 → stdout → cron捕获 → QQ Bot
# API密钥 → profiles/investment/.env 中的 DEEPSEEK_API_KEY
```

**常见错误**：编辑了 `/opt/data/scripts/run_closing.py`，但cron实际执行的是 `profiles/investment/scripts/run_closing.py`。

## 全量审计规则

修改任何cron脚本时，必须检查全部21条定时任务，不能只改相关的几条。
用户会先后问"只有5条吗？""为什么不是全部？"—默认全量覆盖。

## R2 HTML规范

每次上传文档到R2时，必须同时上传MD和HTML双版本。
HTML必须内嵌markdown内容（Python侧转义后嵌入JS模板字面量），禁止前端fetch：
```python
escaped = md.replace('\\', '\\\\').replace('`', '\\`')
html = f'<script>const md = `{escaped}`;</script>'
```
