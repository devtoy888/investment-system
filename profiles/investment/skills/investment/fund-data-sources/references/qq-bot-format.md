# QQ Bot Markdown格式约束

## 核心结论

QQ Bot (api.sgroup.qq.com) 的 `msg_type=2` (markdown) 模式**只支持有限子集**。
以下语法会**破坏显示**，绝对不要用。

## ❌ 不可用的语法

| 语法 | QQ Bot渲染效果 | 应替代为 |
|:-----|:---------------|:---------|
| `## 标题` (H2) | **超大号加粗标题** | `📌 **标题**` — 但**也是大号粗体**，建议直接用 `📌 标题` |
| `### 标题` (H3) | 大号粗体 | 同上 |
| `**粗体**` | **偏大的字号，但可用于标题分段** | 适合做章节标题+emoji组合，正文不用 |
| `#tag` (H1) | **巨大字号**，一整行加粗 | 直接写 `tag` 不加# |
| `_斜体_` | 渲染异常 | 直接写文字 |
| `~~删除线~~` | 不支持 | 不用 |
| `\n\n\n` (多个空行) | 超大段落间距 | 最多 `\n\n` 一个空行 |
| `---` (分割线) | 奇怪的分隔符 | `• • •` 或 `══════` 或跳过 |

## ✅ 可用的

| 语法 | 效果 | 说明 |
|:-----|:------|:------|
| 纯文字 | ✅ 正常 | 最好用 |
| Emoji | ✅ 正常 | 用emoji充当视觉分隔 |
| 表格 | ✅ 正常 | 支持标准markdown表格 |
| 换行 `\n` | ✅ 正常 | 单换行正常 |
| 段落 `\n\n` | ✅ 正常 | 一个空行分段 |
| 短链接 | ✅ 正常 | 自动可点击 |

## PDF/长消息处理

`send_markdown_in_chunks` (send_qq_bot.py):
- 超3800字自动分段
- 接续标记用 `[📎 接上条]` (不用 `_(接上条)_`，因为`_`被渲染为斜体)
- 标题头用 `📚 周度复盘` (不用 `## 📚 周度复盘`)

## 验证方法

```python
# 发送前预览
text = "🔮 下周关注\n\n唐史主任：AI见底\n- ICU KTV"
print(text)

# 检查是否有禁忌语法
import re
bad = []
if '##' in text: bad.append('H2标题')
if '**' in text: bad.append('粗体')
if re.search(r'^\s*#\w+', text, re.M): bad.append('H1标签')
if '___' in text or '_(接' in text: bad.append('斜体标记')
if bad: print(f"⚠️ QQ Bot格式警告: {bad}")
```

## 🚨 链接格式陷阱（2026-07-21发现）

**QQ Bot 不支持 Markdown 链接语法。** `[标题](url)` 格式中的括号 `()` 会被QQ作为URL的一部分，导致点击时带括号→404。

### ❌ 错误
```
📄 [Markdown报告](https://.../report.md)    → 用户点击后跳转到 https://.../report.md)  → 404
🌐 [HTML预览](https://.../report.html)      → 同样问题
```

### ✅ 正确
```
📄 https://.../report.md     → 用户可正常点击
🌐 https://.../report.html   → 正常跳转
```

## 📐 QQ消息 vs R2报告 双输出模式（2026-07-21确立）

所有定时任务报告输出遵循**两层设计**，不可混淆：

| 维度 | QQ消息 | R2报告 |
|:-----|:-------|:-------|
| 定位 | **待办清单/概览** | **完整报告/翔实** |
| 数据 | 精简快照（4行） | 完整行情表+板块+量价+北向 |
| 操作 | 只列有动作的基金(3-5条) | 全部14支基金+策略标签 |
| AI | 一句话理由(≤150字) | 完整分析(步骤1-6) |
| 格式 | **粗体标题+emoji+纯文本URL | MD+HTML双格式 |
| 链接 | 纯文本URL（不可用Markdown链接） | 标准相对路径 |
