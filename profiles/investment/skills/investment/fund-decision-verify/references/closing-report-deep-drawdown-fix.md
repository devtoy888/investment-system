# 收盘复盘 深套约束 + 数据链路修复

## 问题（2026-07-21）

收盘复盘AI分析建议用户对024418（半导体材料设备ETF联接C）"减仓止盈"，但该基金从高点回撤-33.3%（深套）。AI看到了今日+15.8%的涨幅，没意识到持仓处于深套状态。

## 原因链

```
CLOSING_PROMPT_V2 无持仓状态描述
  → AI不知道基金回撤幅度
    → 仅凭当日涨跌做判断
      → 建议"止盈"（完全脱离持仓实际）
```

## 修复

### 1. Prompt层面 — 增加深套约束

在 `llm_analysis_v2.py` 的 CLOSING_PROMPT_V2 步骤3后插入：

```
⚠️ 【持仓状态约束 — 必须遵守】
对于回撤超过20%的基金，严禁建议"止盈/减仓/卖出"。
只允许："持有等待反弹至-15%以内再评估" 或 "继续持有，观望"
对于回撤超过25%的基金，只允许「持有不动」。
```

### 2. 数据层面 — 注入回撤数据

`build_closing_data_v2()` 需要为每支基金提供：
- 昨日净值（nav）
- 今日估算涨跌（estimated_change）
- 估算净值（nav × (1+est_change%)）
- 持仓回撤（drawdown，来自get_history()）

### 3. 展示层面 — 修复估算净值 None

`closing_review.py` 在生成持仓表前手动补全：
```python
fd['estimated_nav'] = round(nav * (1 + ec / 100), 4)
```

同样处理 `llm_analysis_v2.py` 的 `build_closing_data_v2()`。

## 验证方法

```python
# 检查是否有违反约束
if '止盈' in analysis and '024418' in analysis:
    print("❌ AI违反深套约束")
```

完整检查清单：
1. 渲染HTML → 确认基金名显示为代码+全名（非空）
2. 估算净值列 → 确认不是 None（有计算值）
3. 024418涨跌 → 确认是+15.80%（非-6.51%）  
4. AI分析 → 确认024418建议是"持有/观望"（非"止盈"）
5. 全文None计数 → 确认0次（非14次）
