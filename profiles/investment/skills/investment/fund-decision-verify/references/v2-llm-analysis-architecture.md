# v2 LLM分析架构

> 2026-07-20 核心架构文档。覆盖：多数据源注入、T+1约束框架、金融理论原则、六步分析流程。

## 数据流

```
┌─ run_closing.py (cron no_agent)
│   ├─ closing_review.py       → 采集数据 → /tmp/fund_data/
│   ├─ send_closing.py          → 格式化数据表格
│   └─ llm_analysis_v2.py
│       ├─ T1_FRAMEWORK        → 操作约束（15:00/净值未知/T+2）
│       ├─ FINANCIAL_THEORY     → 分析原则
│       └─ 具体prompt          → 分析指令
│           └─ data builder    → 注入七组数据
│
└─ stdout → QQ Bot推送
```

## 六个报告类型的prompt层次

所有prompt通过 `T1_FRAMEWORK + "\n" + FINANCIAL_THEORY_FRAMEWORK + "\n" + 具体指令` 注入约束。

| 报告 | 时间 | 逐基金建议 | 数据量 |
|:----|:----|:----------:|:------:|
| closing | 16:00 | ✅ 明日方向 | ~6000字 |
| morning | 09:00 | ✅ 关注3支+警惕3支 | ~6000字 |
| noon | 11:35 | ✅ 赛道级方向 | ~6000字 |
| decision | 14:30 | ✅ 14支逐一评分+操作 | ~6300字 |
| weekly | 周日 | ✅ 调仓+对比优化 | ~6000字 |
| weekend | 周六 | ✅ 周一预案 | ~6000字 |

## T+1规则（用户纠正版）

**之前错误：** 把T+1理解为"今天决定明天生效"。❌

**纠正后：** 核心是**买入卖出按当日收盘净值计算，净值未知**。
- 15:00前下单 → 按今日收盘净值成交（净值收盘后公布，下单时未知）
- 15:00后下单 → 按下个交易日净值成交
- 份额T+1到账，赎回资金T+2到账（不是T+1！）
- "盲操作"——知道方向但不知道精确成交价

## 核心教训：不要写死规则

用户明确批评在prompt中硬编码规则。正确做法：
1. 给LLM完整数据（市场+持仓+P&L+趋势+新闻+KOL）
2. 框架只提供分析视角（趋势/风险/仓位/再平衡）
3. 让LLM基于数据自行判断，不套固定百分比
