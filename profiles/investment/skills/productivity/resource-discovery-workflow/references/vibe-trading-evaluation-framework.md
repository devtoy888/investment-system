# 开源项目10维度全量评估框架

从 Vibe-Trading (HKUDS, 22.9k stars) 评估中提炼。

## 评估维度表

| # | 维度 | 分析内容 | 源码探测路径 |
|:--|:-----|:---------|:------------|
| 1 | **整体定位** | 解决什么问题？目标用户？股票/基金/加密/研究？ | README + wiki |
| 2 | **交易模型** | T+0 vs T+1？个股 vs 基金 vs ETF？费用结构？ | 引擎代码 + 文档 |
| 3 | **数据源** | 列出所有 loader/数据接口，可用性+接入成本 | `loaders/` `sources/` 目录 |
| 4 | **因子/信号系统** | 算子可复用度（时间序列算子可挪用到净值序列） | `factors/` `signals/` |
| 5 | **回测/模拟引擎** | 配置系统、基准对比、费用模型、滑点处理 | `backtest/` `engine/` |
| 6 | **安全模型** | Mandate/Kill Switch/沙箱/权限分级 | `security.py` `guard.py` |
| 7 | **测试体系** | 测试数量、覆盖率、CI 流程、代码质量 | `.github/workflows/` `tests/` |
| 8 | **工具/API系统** | MCP 工具、CLI、REST API、Agent 集成 | `tools/` `api/` 目录 |
| 9 | **复盘闭环** | Shadow Account、行为诊断、归因分析 | `shadow_account/` `analysis/` |
| 10 | **前端/报告** | Web UI、报告渲染、日志可视化 | `frontend/` `ui/` |

## 评估输出格式

每个维度给出三项：

```markdown
### 数据源（示例）
**可用性评级**: ⭐⭐⭐⭐ 5个源可直接接入
**代码位置**: `sources/` 目录，15个loader
**落地工作量**: 1人天（核心2个接入，其余可后续）
```

### 综合评级

- 🔴 **可直接导入** — 代码复用，最小修改
- 🟡 **需适配改造** — 核心逻辑可借鉴，需改接口
- ⚪ **架构参考** — 不能复用代码，但设计理念可学
- ❌ **不推荐** — 与目标场景根本差异（如 T+0 vs T+1，个股 vs 基金）

### 产出的三个层次

1. **速评**（第一印象）— 30分钟内，基于 README + 目录结构
2. **标准评估**（1-2小时）— 核心源码分析 + 10维度对照
3. **全量评估**（3-4小时）— 加上部署尝试 + 边缘 case 测试
