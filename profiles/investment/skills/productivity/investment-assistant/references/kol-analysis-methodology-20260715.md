# KOL 分析方法论（2026-07-15修订：精简版）

> 基于用户明确反馈的修正：不逐条分析黑话，今日为主历史为辅，去互动数，评论低权重。

## 核心变更对照

| 旧行为 | 新行为 | 用户反馈 |
|:------|:------|:---------|
| 每条微博都跑 `interpret_weibo()` | 仅今日信号帖才跑 | "不要每条微博都分析他的黑话" |
| 所有帖子逐条展示 | 今日帖主分析，历史帖仅做趋势参考 | "近几日的微博是为了帮助今日说的微博分析一致性和趋势" |
| 输出互动数（🔄💬❤️） | 不输出 | "评论点赞数无用，去除" |
| 评论分析含板块讨论+收益自报 | 仅提取主任回复，最多3条 | "评论权重不大，仅做补充" |
| `generate_kol_deep_analysis()` 全局黑话破译 | 已移除该调用 | 全局分析噪音太大 |

## 采集规则

- 每次拉15条/博主
- 用 `_is_today_post()` 分离今日/历史
- 所有帖子均做 `sector_analysis`（赛道+方向匹配），用于趋势一致性

## 信号帖判定

```python
SIGNAL_TRIGGERS = ['加仓','减仓','清仓','抄底','反弹','底部','泡沫','风险',
                   'ICU','KTV','IPO','放量','缩量','格局','右侧','机会','警惕',
                   '止损','触底','泡沫','过热','扫货','建仓','离场','逃顶']

is_signal = any(kw in p['text'] for kw in SIGNAL_TRIGGERS) and _is_today_post(p)
```

- 仅 `is_signal=True` 的帖子调用 `interpret_weibo()`（黑话破译）
- 历史帖不做 `interpret_weibo()`，不输出逐条解读

## 输出格式（_kol_summary.txt）

```
📊 **赛道情绪汇总**（所有帖子，看一致性）
📋 **今日观点 (N条)**（含黑话破译+赛道标签）
📈 **近期趋势参考 (最多5条)**（仅文本+赛道标签，无解读）
```

**不输出：** 互动数、完整帖子列表、逐条黑话解读、板块讨论文本

## 评论补充

- 仅对今日信号帖触发，只拉唐史主任
- 最多15条评论，只提取有主任回复的（最多3条）
- 不分析板块讨论文本、不分析收益自报
- 输出到 `_kol_comment_insights.txt`，不在推送中独占卡片
