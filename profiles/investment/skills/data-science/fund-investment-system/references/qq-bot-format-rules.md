# QQ Bot 推送格式规范（2026-07-19 定稿）

> 从飞书卡片过渡到QQ Bot后，多次格式问题修复的经验总结。

## 格式铁律

| 规则 | 说明 | 修复时间 |
|:-----|:------|:--------:|
| ❌ 不用 `##` 标题 | QQ Bot渲染为超大字体，有违手机阅读体验 | 07-19 |
| ❌ 不用 `**KOL名**` 加粗 | QQ Bot可能渲染为H3标题 | 07-19 |
| ❌ 不输出 `#话题#` 原文 | 微博原文的#号会被QQ Bot解析为标题或乱码 | 07-19 |
| ❌ 不用 `━` `═` 框线字符 | QQ Bot可能渲染异常，换 `• • •` 或空白行 | 07-18 |
| ✅ 用 `▸ KOL名` 列表形式 | 渲染为普通文本列表，不占标题样式 | 07-19 |
| ✅ 操作建议表格放最前 | 用户最关心的是"今天要不要操作" | 07-19 |
| ✅ 4000字符截断 | send_qqbot._output() 内置保护，段落边界截断 | 07-14 |
| ✅ 分片推送（超长内容） | send_qq_bot.send_markdown_in_chunks() 自动分片 | 07-15 |

## 推送内容布局（按重要度排序）

```
1. 今日操作建议（表格: 赛道/建议/置信度/涉及基金）  ← 最前
2. 大盘走势/外盘（表格）
3. 量价分析（表格）
4. 板块排行（表格）
5. 持仓表现（表格）
6. KOL信号摘要（含✅/❌事实核查标记）
7. 统计信息（信号数/准确率）
```
