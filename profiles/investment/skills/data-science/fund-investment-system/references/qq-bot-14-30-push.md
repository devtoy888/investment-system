## 设计原则

1. **紧凑 4 段式** — 止损信号 / 组合偏离度 / 风险预警 / 基准对比，每段不超过8行
2. **Emoji 分级** — 🚨紧急 > 🔴警告 > ⚠️注意 > 🟡关注。一眼识别严重程度
3. **进度条** — `█████████████████░░░` 直观显示占比，每5%一格，共20格
4. **无表格** — 纯 bullet 列表 + emoji，QQ Bot 渲染稳定（表格在QQ mobile易变形）
5. **总长 ≤ 1000 字符** — QQ 单条4000上限，此格式约600-900字符
6. **分隔线** — `━━━━━━━━━━━━━━━━━━━━` 分隔段落（非 markdown `---`，后者在部分平台渲染为断裂线）

## 各段数据来源

| 段落 | 脚本 | 输出位置 |
|:----|:-----|:---------|
| 止损信号 | `signal_engine.py` | stdout → `run_buy_signal.py` |
| 组合偏离度 | 直接计算 `PORTFOLIO_COST` | `format_op_push.py` 内联计算 |
| 风险预警 | `risk_warning.py` → 过滤 >5%暴跌 | stdout → `run_buy_signal.py` |
| 基准对比 | `check_benchmark.py` | stdout → `run_buy_signal.py` |

## 关键实现 (format_op_push.py)

- `run_script(name, timeout)` — 子进程调用，超时保护
- 偏离度：从 `log_daily_decisions.PORTFOLIO_COST` 读取持仓，`get_fund_value()` 静默获取→每组 totol_now / total → 占比%
- 超配警报：科技/AI >50%🚨, >30%⚠️
- 基准对比：从 `check_benchmark.py` stdout 提取沪深300 和 组合涨跌幅行

## Pitfalls

1. **`get_fund_value()` 的 stdout 干扰** — fund_tools 会打印 `✅ 基金名: 净值=xx 估算=xx%`。在偏离度计算时必须重定向 stdout：`sys.stdout = io.StringIO()` → `get_fund_value(code)` → `sys.stdout = old_stdout`
2. **signal_engine.py 也打印日志行** — 需过滤掉 `✅ 开头` 的行后才能提取止损信号
3. **偏离度百分比计算要用 PORTFOLIO_COST 的 cost_total** — `group_funds()` 返回的 `total_pct` 等字段可能不存在。正确的路径是：PORTFOLIO_COST → get_fund_value(静默) → now_val = cost * (1+est/100) → sum by group
4. **14:30 cron 当前为 30 14 * * 1-5** — 不要随意改动
