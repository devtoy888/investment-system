# QQ Bot Markdown 局限性（2026-07-19 定型）

> QQ Bot `msg_type=2` 的 markdown 渲染器有严格限制，与 GitHub Flavored Markdown 差异很大。
> 所有投资类/财经类推送都必须遵守此文档的规则。用户因格式问题纠正过5次以上。

## 核心原则

**QQ Bot 只安全支持：emoji + 纯文本 + 表格。** 其他所有 markdown 语法都可能渲染异常。

## 语法兼容性矩阵

| 语法 | QQ 效果 | 风险等级 |
|:-----|:--------|:--------:|
| `文本` 纯文字 | ✅ 正常 | 安全 |
| `emoji` 📈📉🔴🟢🟡 | ✅ 正常 | 安全 |
| `| 表格 \|` | ✅ 正常 | 安全（列数≤6） |
| `**粗体**` | ⚠️ 渲染为超大号文字 | 🔴 高风险 |
| `## 标题` | ⚠️ 渲染为超大号标题 | 🔴 高风险 |
| `#话题` (行首) | ⚠️ 被解析为 H1 标题 | 🔴 高风险 |
| `_斜体_` | ⚠️ 渲染异常 | 🟡 中风险 |
| `> 引用` | ⚠️ 可能渲染异常 | 🟡 中风险 |
| `---` 分割线 | ⚠️ 可能不显示或异常 | 🟡 中风险 |
| `[链接](url)` | ⚠️ 桌面版可点，手机受限 | 🟡 中风险 |
| ``` 代码块 | ⚠️ 渲染不可预测 | 🟡 中风险 |
| `- 列表` | ⚠️ 间接可用但间距大 | 🟡 中风险 |

## 禁止规则

| # | 规则 | 错误示范 | 正确做法 |
|:-:|:-----|:---------|:---------|
| 1 | ❌ 不用 `#` 或 `##` 标题 | `## 大盘走势` | `📈 大盘走势` |
| 2 | ❌ 不用 `**bold**` | `**科技/AI**` | `科技/AI` |
| 3 | ❌ 不展示 `#话题` 原文 | 微博 `#ICU#` 原文 | 清理为 `ICU` |
| 4 | ❌ 不用 `_italic_` | `_(接上条)_` | `[接上条]` |
| 5 | ❌ 不用长框线 | `━━━━━` 或 `════` | `• • •` 或空行 |
| 6 | ❌ 不多余空行 | `\n\n\n\n` 段间距 | 最多 `\n\n` 一个空行 |

## 清理微博文本中 Markdown 的方法

```python
import re
# 一行清理所有标记
text = re.sub(r"[#*_~`]", "", text)
# 未清理的后果：
# "#ICU#KTV" → QQ 渲染为 H1 标题 "ICU#KTV"（超大号字）
# "**下周关注**" → QQ 渲染为超大号文字
```

## 推送格式最佳实践

### 操作建议表格（最前面）

用户最关心的信息放最前面，用表格格式：

```
📋 周一开盘操作清单
| 优先级 | 方向 | 基金 | 代码 | 仓位调整 |
|:-----:|:---:|:----|:----:|:-------:|
| P1    | 📈  | 华夏科创50ETF联接C | 011613 | 5% |
| P2    | 📉  | 大摩资源优选混合 | 163302 | 3% |
```

### Section 标题用 emoji + 纯文字

```
📊 组合周度检视
📈 趋势诊断
💬 KOL信号聚合
📋 周一操作清单
📅 下周关键节点
```

### 信号验证结果

```
📉 科技/AI #bearish #today: ✅ 正确(实际科创50涨跌-7.12%)
原文引用: 「市场在形成共识之前，还会处于高波的状态...」
```

## 验证方法

推送前在 debug 模式下输出纯文本版本检查：
```python
# 检查 markdown 禁用语法
if '## ' in text or '# ' in text or '**' in text:
    logger.warning("推文含禁用markdown语法")
```
